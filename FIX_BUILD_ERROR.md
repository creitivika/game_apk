# Исправление ошибки LT_SYS_SYMBOL_USCORE

Ошибка возникала во время сборки `libffi`: системе не хватало макросов GNU libtool из пакета `libltdl-dev`.

В исправленной версии:

- установлен `libltdl-dev`;
- добавлены остальные системные зависимости из актуального окружения Buildozer;
- используется Java 17 и Python 3.12;
- локальная папка `.buildozer` очищается перед сборкой, чтобы не использовать незавершённые файлы прошлой попытки;
- кэшируются только скачанные Android-пакеты и Gradle;
- сборка выполняется для `arm64-v8a`, что подходит современным Android-устройствам и сокращает время сборки;
- workflow находится в `.github/workflows/main.yml`.

## Что сделать в существующем репозитории

1. Заменить файл `.github/workflows/main.yml` файлом из этого архива.
2. Заменить `buildozer.spec`.
3. Сохранить изменения в ветке `main`.
4. Открыть `Actions` и запустить `Build Android APK` повторно.
5. Старый неудачный запуск удалять не обязательно.

После зелёной галочки APK появится в разделе `Artifacts` под названием `digital-pet-apk`.
